20W vs Other Low-Power Options

Not all low-wattage devices are equal for AI inference. The choice of silicon matters as much as the wattage ceiling:

Raspberry Pi 5 (5–12W)

Excellent for IoT and sensor work, but the ARM Cortex-A76 CPU has no dedicated NPU for large model inference. Running a 7B LLM on Pi 5 is possible but extremely slow — typically 0.5–2 tokens/second. Suitable for small models (Phi-2, TinyLlama) in applications where latency isn't critical.

Intel NUC with integrated graphics (15–45W)

The Intel Arc GPU in newer NUCs can offload some inference, and OpenVINO helps, but you're still far behind dedicated edge AI silicon. Power draw spikes under load. Windows licensing adds cost. Better for general computing that occasionally runs AI.

Apple Silicon Mac Mini M4 (10–25W)

Exceptional unified memory bandwidth for LLM inference — the M4 handles 13B models smoothly. Higher cost (€799+) and macOS licensing. If you're already in the Apple ecosystem, this is a strong option. ClawBox targets the Linux-native, open-source AI stack.
